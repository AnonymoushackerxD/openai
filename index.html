Description

The subdomains comp-calculator.people.openai.org and comp-calculator-admin.people.openai.org are configured as CNAMEs pointing to an Azure Static Web Apps endpoint which currently returns the provider default 404 page. The DNS and HTTP responses confirm that the target application is not provisioned, making these subdomains susceptible to takeover if the corresponding Azure resource is re-created by an attacker.

Evidence

$ dig +short CNAME comp-calculator-admin.people.openai.org
comp-calc-admin-deezaqaafyg5b4h4.northcentralus-01.azurewebsites.net.

$ dig CNAME purple-flower-04197221e.5.azurestaticapps.net +short
azurestaticapps5.trafficmanager.net.

$ curl -I https://purple-flower-04197221e.5.azurestaticapps.net
HTTP/2 404
content-type: text/html

Steps to Reproduce (Non-Invasive)

  1. Check the CNAME for the host: 
    dig +short CNAME comp-calculator-admin.people.openai.org
  2. Query authoritative nameserver for confirmation: 
    dig @ns1-09.azure-dns.com comp-calculator-admin.people.openai.org CNAME +noall +answer +authority
  3. Request the host to observe Azure 404 default page: 
    curl -I https://purple-flower-04197221e.5.azurestaticapps.net
  4. Verify no content hosted or claimed under the domain.

Targets

  • comp-calculator.people.openai.org
  • comp-calculator-admin.people.openai.org
